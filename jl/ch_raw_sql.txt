[Heading2] SQL の基本構文
SQL  は人工的な言語であり，構文や単語が定義されています．最も単純な形は以下のようになります．
SELECT (項目，式)FROM (表)WHERE (条件)SELECT (項目，式)FROM (表)WHERE (条件)

FROM (表)

SELECT，FROM などの英語が SQL で登録された単語であり，( )内が指定する要素名です．文は上から下へと順に読みますが，改行は必ずしも必要ではなく，1 行にまとめても構い
ません．
たとえば，図のような学籍簿と成績表の 2 つの表が与えられたとします．
このとき，出身地のリストを抜き出す射影を実現する SQL 文は次のようになります．

す．
「東京都」出身の人のデータを抜き出す選択の操作は次の SQL 文で実現でき，右のような結果が得られます．

の中身が「東京都」に等しいという条件を表しています．「東京都」は文字列であるということを表すために「'」(シングルクォート)で囲みます．
通常， SELECT の後にはフィールド名を書きますが，フィールド名を用いた式を記述することもできます． たとえば， 次の文では「英語」と「数学」の合計点を抜き出すことができます．

[Heading2] 並べ替え
射影の結果を並べ替えるには ORDER BY という単語を使います．昇順(小さい順)の場合は ASC，降順(大きい順)の場合には DESC を記述します．ASC は ascend(上る)，DESC は descend(下る)の略です．
SELECT 氏名, 英語 FROM 成績表 ORDER BY 英語 ASCSELECT 氏名, 英語 FROM 成績表 ORDER BY 英語 ASCSELECT 氏名, 英語FROM 成績表ORDER BY 英語 DESCSELECT 氏名, 英語FROM 成績表ORDER BY 英語 DESC

[Heading2] 結合
結合の操作は 2 つ以上の表を使います．コンピュータでは表のどの部分が一致しているのかを指定する必要があるため，次の例のように  WHERE の条件文で「学籍簿.学生証番号 =
成績表. 学生証番号」としています．「学籍簿. 学生証番号」というのは，「学籍簿」の表の中にある「学生証番号」というフィールドを表します． 単純に「学生証番号」と書くと，
「学籍簿」のフィールドか，「成績表」のフィールドか区別できないため， 表の名前を明示しています． SELECT 文においても，「氏名」のフィールドが 2 つの表にあるために「学籍
簿.氏名」のように指定しています．
SELECT 学籍簿.氏名, 英語, 出身地FROM 学籍簿, 成績表WHERE 学籍簿.学生証番号＝成績表.学生証番号SELECT 学籍簿.氏名, 英語, 出身地FROM 学籍簿, 成績表WHERE 学籍簿.学生証番号＝成績表.学生証番号
