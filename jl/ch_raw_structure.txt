[Heading1] コンピュータの構造(Cấu trúc máy tính)
パソコンのカタログを見てみると，仕様の欄には細かい数字や聞き慣れない用語が並んでいます． これらは， パソコンのハードウェアの性能を表しています． パソコン内部を目にする機会はあまりないかもしれませんが，中身についての概要が理解できれば，これらの数字の持つ意味が分かってきます．この章では，ハードウェアの構成や特徴について学習します．
[Heading2] コンピュータの構成要素
コンピュータは，演算装置，記憶装置，制御装置，入力装置，出力装置の 5 要素からなります（ 図 52 ）．演算装置は， コンピュータの頭脳にあたる装置で，計算などの処理を行います．CPU と呼ばれる装置が演算装置の機能を持ちます．記憶装置は，さまざまなデー
タを記憶しておく部品で， メモリやハードディスクなどが該当します． 制御装置は， 記憶装置からデータを読み込み，演算装置の指示に従って，演算結果を記憶装置に書き込んだ
り，制御信号を送ったりします． CPU にも制御装置の機能があります． 入力装置は，ユー
ザからデータ入力を受ける装置で， キーボードやマウスなどがあります． 出力装置は， 結果をユーザに提示する装置で，ディスプレイやプリンタなどがあります．

[Heading2] パソコンの内部構成
デスクトップパソコンの蓋を開けてみると図 53 のように様々な部品を見ることができます． なお， 蓋を開けるときには電源コードを抜き， 感電に気を付けるとともに， 自分が
帯電して電子部品を壊すことがないように，地面につながる金属部分に触れておきましょう． また， ノートパソコンは， 分解するとメーカ保証が受けられなくなることがあるので注意してください．
図 53 コンピュータ内部
パソコン内で大きな面積を占めるメインとなる基板のことをマザーボードと呼びます
（図 54）．マザーボード上で，ファンか放熱板が付いている部品が CPU です．CPU とは Central Processing Unit の略で，中央演算処理装置と訳されています．処理の中心となる部品であり，コンピュータの頭脳などとも言われます． CPU の中には演算装置と制御装置の機能が組み込まれています．DVD 鑑賞や高品位な 3 次元ゲームに高度な機能が求められるため，CPU  とは別に画面表示専用のグラフィックプロセッサ(GPU)が搭載されていること
もあります．
いくつかの IC が規則正しく並んでいるものがメモリで，記憶装置として働きます．CPU
からのデータを直接受ける記憶装置なので主記憶装置とも呼ばれます．
マザーボード上での CPU，メモリ間などの電気の通り道のことをバスと言います．バスを流れるデータや外部機器とのデータを制御する IC の組をチップセットと言います．
[Heading9] 図 54 パソコン内部のデータの流れ
マザーボードとケーブルで接続された箱がハードディスクです．ハードディスクは記憶装置の 1 つですが，メモリの主記憶装置に対して補助記憶装置と呼ばれます．CD-ROM や DVD-ROM などの外部記憶装置もマザーボードからケーブルで接続されています．
パソコン本体内部の部品に対してキーボード， マウスなどの入力装置とモニタ， プリンタなどの出力装置を合わせて周辺機器と呼びます． 多くの周辺機器は USB(Universal System Bus)などの外部インタフェースを介して接続されます．
[Heading2] 記憶装置の特徴
コンピュータとは，記憶して処理する機械ですが，処理する部品が CPU だけなのに対し， 記憶する部品はメモリ， ハードディスクなど多種類あります． また， CPU 内部にもキ
ャッシュと呼ばれる記憶領域があります．記憶装置がたくさんあるのは，それぞれの装置に特徴があるからです．キャッシュは，データのアクセス速度が最も高速ですが，記憶容量が非常に小さい記憶装置です．メモリは， キャッシュよりは記憶容量が大きくなりますが，アクセス速度はキャッシュより遅くなります． ただし， キャッシュと同様に電気的な読み書きなので比較的高速です．ハードディスクは，大容量の記憶が可能ですが，機械的な読み書きのためアクセス速度は遅くなります．一般に，アクセス速度と記憶容量は反比例の
関係になります（図 55）．
[Heading9] 図 55 アクセス速度と記憶容量の関係
[Heading2] 半導体メモリ
半導体メモリ，もしくは単純にメモリと呼ばれるものには RAM とROM があります．
RAM(ラム)とは Random Access Memory の略で，自由にデータの読み書きができる半導体メモリです． RAM は電源が切れるとデータも消えてしまうため， 短期的記憶にしか使えません．一般にパソコンのメモリとして使われているのは RAM のうち DRAM(Dynamic RAM)と呼ばれているもので，静電気を記録する単純な構造のため安価に製造できますが，
一定時間ごとの再読み込みが必要なため， アクセス速度がやや遅くなります． キャッシュには SRAM(Static RAM)が使われます．SRAM は論理回路を構成する必要があるので高価
ですが，再読み込の必要がないため低消費電力でアクセス速度は高速です．
画面表示用に画面イメージ全体を記憶しておくメモリは VRAM(Video RAM)と呼ばれます．VRAM の記憶容量によって表示可能な解像度や色数が決まります．
ROM とは，Read Only Memory の略で，読み取り専用の半導体メモリです． CD-ROMや DVD-ROM も読み取り専用という意味で ROM という名前になっています．電源を入れた時点では RAM には何も記憶されていないので，コンピュータは何も実行することができません．そこで，システムの立ち上げに必要な最小限の情報を ROM に記憶させることで起動を可能としています．一般の  ROM  は製造工程で情報が書き込まれていて書き換え
はできませんが，紫外線や電気で情報を消去して再度書き込み可能にしたものもあります．
デジタルカメラなどで使うフラッシュメモリも ROM の一種です．
論理的に見ると，メモリは 0 と 1 の並びであるデータを蓄える箱の集まりで，1 つの箱には桁数の決まった 1 つのデータを記録することができます（ 図 56）．固定した桁数の記
憶領域が連続した場所であるメモリ空間には，それぞれの記憶場所に記憶アドレスという番地が割り当てられています．桁数は 32 ビット，64 ビットなどの数字によって決まります．
図 56 メモリの論理構成
ハードディスクの内部では，磁性体を塗った何枚かの金属の円板が 1 分間に数千回転の高速で常に回転しています．円板の円周に直交する方向に動くアームの先端には磁性体の
情報を読み書きするヘッドが取り付けられ，円板との間にわずかの隙間をあけて移動します（ 図 57 ）． ヘッドと円板とは直接接触しないため， アクセス速度を上げることができ，
ディスク面の劣化もありません． しかし， 衝撃を加えるとヘッドがディスク面に接触してしまい，データが読み書きできなくなることがあるので注意が必要です．
図 57 ハードディスク内部
アームを固定したときにヘッドがディスク上で描く軌跡をトラックと言います（ 図 58 ）． 記憶領域を管理するときにはトラックをセクタと呼ばれる等分割の領域に分けます．論理的
なアドレスは， トラックとセクタで表される物理的なアドレスに内部的に分解されてアクセスされます． 同一トラックにデータがあれば， アームを動かさなくて済むので高速な読
み書きができます． Windows などにある「ディスクの最適化」という操作は， よく使うフ
ァイルを同一トラック， もしくは近くのトラックにまとめることで， 無駄なアームの動きを無くし，高速化をはかる機能です．この操作のことをデフラグメンテーションと言います．
[Heading9] 図 58 ハードディスクの記憶領域
[Heading2] CPU
CPU には Intel 社の Core i7，Core 2 Duo，ATOM などの種類があります．なお，CPUは機能として見たときの呼び方で， ハードウェアとして見たときにはマイクロプロセッサ，
または MPU(Micro Processing Unit)と呼ばれます．
CPU の内部には演算装置と制御装置があり，バス(内部バス)を通じてデータのやりとりをします． 1 回に何桁のデータをバスで送ることができるかで， 32 ビットの CPU， 64 ビットのCPU などと呼ばれ，この数字が大きいほど効率良く計算できます．
CPU  には命令セットと呼ばれる単純な演算処理，制御処理が実行できますが，CPU  の種
類が違うと実行できる処理は異なります．共通に使える命令の代表的なものは，データの読み書き，足し算，引き算，0 と 1 の判断，アドレスジャンプなどです．
命令はクロックという単位で動作します． クロックとは， CPU やメモリなどのすべての部品の同期をとる時計のことで，クロック信号が送られたときに，一斉に 0 か 1 かを判断します．たとえば，足し算は 1 回のクロックで終了，かけ算は 2 回のクロックで終了というように， 命令は何クロックで終了するかが決められています（ 図 59 ） ． したがって， 1
秒あたりのクロック数が大きくなればなるほど高速に計算できることになります．
図 59 CPU の命令実行タイミング
1 秒あたりのクロック数をクロック周波数と言い， ヘルツ(Hz) の単位で表されます．たとえば，1 クロックで終了する命令は，1 GHz(ギガヘルツ=1,000,000,000 ヘルツ)の CPU であれば，1 秒間に 10 億回の命令が実行でき，クロック周波数が高くなればなるほど高速に計算することができます． ただし， これは同一 CPU で比較した場合だけで， 命令セットの異なる CPU の場合には，処理内容によって得意，不得意があるので，クロック周波数
だけで計算が高速かどうかを知ることはできません．
クロック周波数を高くすると， 消費電力が増加するため， バッテリー駆動時間が短くな
ります． また， 発熱量が増えるため， 冷却対策が必要になります． このため， 最近ではクロック周波数を高くするのではなく， CPU の演算装置を多重化することで高速化しています．たとえば，1 個の演算装置を持つ Core Solo という CPU に対して，Core Duo は 2 個， Core Quad は 4 個の演算部分があります．ただし，演算装置が 2 個でも計算速度は 2 倍に
は至りません．
単純に整数演算の性能，実数演算の性能，グラフィック描画の性能などを比較するためには実際の計算時間を計測します．計測の単位には， 1 秒あたりに実行できる命令回数である MIPS(Million Instructions Per Second)や，1 秒あたりに実行できる実数演算の回数である FLOPS(FLoating point number Operations Per Second)などを使います．同一プ
ログラムによって性能を計測するテストをベンチマークテストと言います．
