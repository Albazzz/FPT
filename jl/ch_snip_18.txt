[Heading1] プログラミングの基礎(Cơ sở lập trình)
コンピュータのソフトウェアにはワープロ， 表計算などがありますが， これらは， プログラムという命令の集まりによって記述されています． プログラムの基本は比較的単純であり， それを論理的に組み立てていくことで複雑な作業を実現することができます． この章では， コンピュータソフトウェアを作成するためのプログラミングの基礎知識について学習します．
新しい言葉
[Heading2] プログラミング言語
プログラミング言語とは，コンピュータに対する命令記述の集まりであって，人工的な文法規則を持ちます．たとえば，SQL もプログラミング言語の 1 つと考えられます．
プログラミング言語には多くの種類があり，図 104 のようなものが代表的です．
図 104 代表的なプログラミング言語
これらは大きく分けると手続型と非手続型に分けることができます． 手続型言語は， 処理の手順を与えて， 何をどうするか(How) を詳細に記述する言語であり， Java ， C++など多くの言語が属します． 非手続型言語は，何をしたいか(What)を記述することで解を得る言語で，SQL など目的の特化された定型的処理をする少数の言語が属します．
C++言語は汎用性が高いので科学技術を始めとする一般的な用途で使われますが，生成されたプログラムは  Windows  用，MacOS  用のように，利用する  OS  ごとに用意しなければなりません．Java  言語は，文法的には  C++に似ていますが，OS  に依存せずに  Windows， MacOS，携帯電話などで同じように実行できます．ただし， C++に比べると実行速度は遅
くなります．
HTML 言語は，ホームページを記述するのに使われます．C++や Java のような計算はできないので，計算や条件判断などの処理をする場合には  JavaScript  言語などと組み合わせ
て記述しなければなりません．XML 言語は HTML を拡張したもので，ホームページ上で
データベースシステムとして利用できるため，ビジネスにおいては重要な言語です．
[Heading2] プログラムの内部動作
プログラムは内部的にはメモリに記憶されますが， メモリ上ではデータもプログラムも区別がありません．
CPU はメモリ上のデータをアドレスに従って 1 つずつ読み込み，逐次的に処理を継続します( 図 105)． メモリに蓄えられているプログラムは， CPU の種類に依存した機械語と
呼ばれる低級言語です．低級言語は，汎用性や読みやすさに問題があるので，プログラムの作成には Java  や C++などの高級言語が用いられます．高級言語で記述したプログラム
は，コンパイラというプログラムで自動的に低級言語に翻訳されます．

[Heading4] 代入

「;」は文と文の区切りを意味します． 直観的には， 変数とは図 107 に示すような Excelのセルに当たる箱であり，箱の名前が変数名です．変数に数字を入れることを代入(だいにゅう) と呼びます． 変数名はアルファベットで始まる名前であり， プログラムであらかじめ

[Heading4] 配列
変数の数が少ないときには変数名を a，b，c のように変えればよいのですが，数千，数万という大量のデータが使われるような場合には変数名を付けることが困難になります．

番号は A1 ， A2 ， A3 のようになりますが， プログラムの配列では， a[0] ， a[1] ， a[2] のように記述します．a という名前の 0 番目，1 番目，2 番目の変数という意味です．[ ]の中の

[Heading4] 条件判断
四則演算以外で最も重要なプログラムの機能の  1  つが条件判断です．条件判断と言って
も複雑な判断ができるわけではなく，真か偽かを簡単に判定できるものに限られ，具体的には等しいかどうかや大小関係の判断になります．
プログラムでは次のような形式になります． 条件としては， Java や C++では右のよう
なものが一般的です．
次のプログラムは a が奇数ならば「奇数」と表示し， 偶数ならば「偶数」と表示する処理を行います．この処理は，整数を 2 で割った余りが 1 ならば奇数，0 ならば偶数という
性質を利用します．3 行目は 2 で割った余りが 1 と等しいかどうかを判断しています．も
しも 1 ならば奇数なので上の{ }内を実行し，そうでなければ下の{ }内を実行します．ここで，「// 」は， この記号の右に書かれたものはプログラムとして実行しないというコメントを表します．本来は，コメントの部分に適切なプログラムを記述しなければなりません．
a = 10;a = a % 2; if (a == 1) {// 「奇数」と表示} else {// 「偶数」と表示}a = 10;a = a % 2; if (a == 1) {// 「奇数」と表示} else {// 「偶数」と表示}
a = 10;
a = a % 2; if (a == 1) {
// 「奇数」と表示
} else {
// 「偶数」と表示
}

}
[Heading4] 繰り返し
条件判断と並んで重要な機能が繰り返しの処理です． これは，ある条件が成り立つ間，決められた処理を繰り返すというものです．形式は次のようになります．
for (初期値; 継続条件; 増分) {実行文}for (初期値; 継続条件; 増分) {実行文}
for (初期値; 継続条件; 増分) {
実行文
}
for (初期値; 継続条件; 増分) {
実行文
}
たとえば，次のプログラムは初期値が i = 0 であり，i が 10 以下の間，i を 2 ずつ増やして i を表示するものです．コメントの部分に i を表示する処理を追加すれば，実行結果
は，0, 2, 4, 6, 8, 10 と表示されます．i = 0 という代入文は，最初に一度だけ実行されま
す．繰り返されるのは{ }の中に書かれた処理になりますが，処理が終わるたびに i = i + 2 という代入文も実行されます．
for (i = 0; i <= 10; i = i + 2) {// i を表示}for (i = 0; i <= 10; i = i + 2) {// i を表示}
for (i = 0; i <= 10; i = i + 2) {
// i を表示
}
for (i = 0; i <= 10; i = i + 2) {
// i を表示
}
[Heading2] オブジェクト指向
手続型の言語では，単純な文法で手順を記述しますが，一度作成したプログラムの再利用を容易にするためにオブジェクト指向という概念が考案されました．
オブジェクト指向とは， 従来のプログラミングのような「こと」を中心にするのではなく，
「もの」を中心にするという考え方です． Java や C++など最近のプログラミング言語で
は，オブジェクト指向に基づいたプログラミングができるようになっています．
オブジェクトとは， 分類された集合に属する「もの」のことであり， この集合をクラスと呼びます． たとえば， 「ヒト」というクラスに属するオブジェクトは， 話したり笑ったりとい
う動作( メソッド) をします． また， ヒトには， 身長，体重など固有の属性( 変数) があります．
変数に固有の値を与えることで，オブジェクトの実体(インスタンス)を作ることができます．たとえば， ヒトについては， 身長 180cm， 体重 70kg の「田中」という名前を持つオブジェクトは 1 つのインスタンスです．
息をする， 眠るなどの動作や， 身長， 体重などの属性はヒトだけでなく他の動物にも共通します． 共通する性質をクラスごとに整理し， そのクラスに固有のメソッド， 変数だけを記述すれば， 記述を大幅に簡略化できます． たとえば， ヒトのクラスは哺乳類クラスに属し， 哺乳類クラスは動物クラスに属しているので， 性質は上位クラスから下位クラスへ
と継承されます( 図 110) ． 継承により， 上位クラスについて知ることなく最小限の違いだ
けを記述できるため， プログラムの生成， 変更が容易になります． また， 情報がカプセル
化されるため， そのクラスでの処理記述が上位クラスに影響を及ぼすことがなくなり， 全体の整合性を保つことができます．
図 110 オブジェクト指向の継承関係
