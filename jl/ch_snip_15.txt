[Heading1] オペレーティングシステム(Hệ điều hành)
新たにパソコンを購入するときの大きな選択は Windows にするか Mac にするかです． Windows のほうが使えるビジネスソフトの数が多く， 一方， Mac のほうがデザイン作業に優れていると言われ， それぞれに一長一短があって選択に迷うところです． Windowsと Mac の本質的な違いとは， ハードウェアの差ではなく， 基本ソフトであるオペレーティ
ングシステムの違いです． この章では， オペレーティングシステムとは何かについて学習します．
新しい言葉
[Heading2] オペレーティングシステムとは
Windows， MacOS，Linux などのソフトウェアは， オペレーティングシステム，または略して OS と呼ばれます．OS はソフトウェアの一種ですが，ワープロや表計算などのアプリ
ケーションソフトと， ハードウェアとの中間的な位置付けになります． このため日本語では基本ソフトと呼ばれることもあります． 一般に， アプリケーションソフトは個別の OS 用に開発されていて，Mac 用のソフトはWindows では使えませんし，逆もできません．
アプリケーションソフト以外のソフトウェアのことをシステムソフトウェアと呼び， OS はその一種です（ 図 94 ）． OS 以外のシステムソフトウェアとしては， 最低限のハードウェア制御を行い，ブート(起動)時に OS をハードディスクからメモリに移動させる BIOS(バイオス) というソフトウェアがあります． また， システムソフトウェアの中で， アプリケーションソフトと OS の中間的な位置付けとして， ミドルウェアと呼ばれるソフトウェアもあります． いくつかのアプリケーションソフトは， 直接 OS 上ではなく， データベース管理シ
ステムやソフトウェア開発支援ツールなどのミドルウェア上で動きます．

[Heading2] OS の機能
以下に OS の機能について説明します．
ユーザインタフェース
OS の機能の第 1 は，操作性を決めることです．すなわち，マウスやキーボードの操作
の仕方， ウィンドウの見え方， ファイルの扱い方などを決定します． この操作性のことをユーザインタフェース と言います． 特に見え方に関することを GUI(Graphical User Interface) と言います． 最近のパソコンでは， ウィンドウ操作という概念により， 使い方はほぼ共通化されてきました．ただし， Mac ではマウスボタンが 1 つ，Windows では 2つ，Unix では 3 つあるのを前提にしていたり，キーボードにも Mac 用の特殊なキーや Windows 用の特殊なキーがあったりするため，操作性に若干違いがあります．
ソフトとハードの仲介
OS の機能の第 2 は，図 95 に示すように，ハードウェアの相違を吸収することです．パソコンに OS が搭載される以前において， アプリケーションソフトは， ハードウェアメ
ーカ毎に対応する製品が作られていました． また， マウスやキーボードなどのハードウェアもメーカ毎に開発されていました． OS によって共通化されることにより， このような開発効率の悪さが改善され， OS が定めた基準を満たしている限り，どのメーカのソフト，
ハードでも問題なく使うことができるようになります．
図 95 OS の位置づけ
記憶管理
OS の機能の第 3 は，記憶の管理です．これはユーザが直接目にするわけではありませんが，とても大事な機能です．「10 コンピュータの構造」では，処理をつかさどる機器が CPU だけであるのに対し， 記憶する機器はキャッシュ， メモリ， ハードディスクなど多数
あるということを述べました． これは， アクセス速度と記憶容量の違いがあるためですが，それぞれの特性に応じて効率よくデータを移動するのは  OS  の役目です．CPU  が処理する
ときには， 可能な限り高速なキャッシュを記憶領域として利用するのですが， キャッシュの記憶容量が足りなくなったところで， メモリとの間で必要なデータを入れ替えます． 同様にメモリとハードディスクの間でもデータを交換し， 常に最も効率がよくなるようにデ
ータを記憶装置に配置します（ 図 96 ）．記憶配置の処理をどのタイミングで行うかによっ
てコンピュータが速く感じられたり遅く感じられたりします．
見かけ上の記憶容量を実際のメモリ容量よりも大きくし， 足りない部分はハードディスクを利用することで， 大きなメモリ空間を確保することを仮想記憶と言い， メモリとハードディスクのデータの交換をスワッピングと言います．
[Heading9] 図 96 記憶装置間のデータ移動
プロセス管理
OS  の機能の第 4  は，プロセスの管理です．プロセスとはソフトウェアの処理のことです．
通常， パソコンではワープロ， 表計算， メールソフト， ブラウザなどが同時に動いているよう見えますが，CPU が 1 つの場合，ある瞬間に計算できるのは 1 つのプロセスだけで
す． しかし， ワープロを入力している間に， 音楽を再生できないというのでは不便です．そこで， OS はアプリケーションソフトのプロセスを細かい単位に分け， それぞれの負荷
に応じて最適なスケジューリングを行うことで， 複数のソフトが同時に動いているように見せています（図 97）．
[Heading9] 図 97 見かけ上の同時処理実現法
複数のプログラムが同時に動くように管理することをマルチタスクと呼んでいます．
ユーザ管理
OS の機能の第 5 は，ユーザ管理です．学校や会社，また家庭内でも同じパソコンを複
数の人が利用することがあります．複数ユーザが利用するときには，ユーザ毎に利用環境を保存し，他人のファイルや情報へのアクセスを制限するユーザ管理機能が必要です．
複数のユーザが利用することをマルチユーザと言います．マルチユーザとは単に複数の人が切り替えて使うだけでなく， ネットワークを介して同時に複数の人が同一のコンピュータを利用しても，適切にプロセスが実行され，他のユーザのファイルや作業を侵害しない安全性，安定性を持つ必要があります．
[Heading2] OS の種類
OS は， 家電製品からスーパーコンピュータまで， 多くの機器で使われています． ここでは，身近にある OS の特徴や歴史について述べます．
UNIX
OS は，もともと大型計算機用に設計され，高機能化に伴って，内部処理が複雑になる
方向に開発が進められました． これに対して，小さいコンピュータでも稼働する簡潔で拡張性が高い OS をめざす動きがあり，AT&T ベル研究所やカリフォルニア大学バークレー校が中心となって Unix という OS が開発されました．Unix(ユニックス)は OS としての性
能が優れていただけでなく，無償で使うことができ， 内部の情報も公開され， 自由に改良することができたため， 世界中の大学を始めとした研究機関で利用されていました． インターネットの普及とともに，より多くの人たちの協力で改良されて発展を遂げました．
その後，ライセンスが企業に渡り，商用の Unix が販売されましたが，開発当初の協力体制，無償奉仕の精神はパソコン用の OS である Linux(リナックス)へと受け継がれることになりました．Linux  はフィンランドのリーナス・トーバルズが  Unix  の機能を独自に実装し，
フリーウェアとして配布したものです． 無料で使えることと， 内部処理が公開されていてセキュリティ上安全であるため，現在でも企業のサーバ等に使われています．
Windows
Windows は，当初ベンチャー企業だったマイクロソフトが開発した MS-DOS という OSが起源です．マイクロソフトはビル・ゲイツが興した小さな会社でしたが， IBM という巨大企業がパソコン市場に参入したときの OS として採用されたため，飛躍的にユーザ数が増えました． MS-DOS は， 画面に文字しか表示されず，入力もすべてキーボードから行うと
いうもので， 一般の人には使いにくかったのですが， その後マウス操作によるウィンドウシステムが利用できる Windows が開発され，飛躍的にユーザ数を伸ばしました．Windows 2000 以降では，中身を大幅に変更することで，動作の安定性を増しています．
MacOS
MacOS はアップル社が開発した OS です．もともとは，スティーブ・ジョブズとスティーブ・ウォズニアックという 2 人が，車庫の工場でパソコンを組み立てていたベンチャー企業でしたが， 1984 年にマウスで操作できるウィンドウシステムという画期的技術をパソコン上で実現した MacOS を開発し，優れたデザインと使いやすさで多くのユーザを獲得しました． MacOS X 以降ではUnix の技術を使い，安定性を増しています．
家電用 OS
OS はパソコンだけでなく， ハードディスクレコーダ， カーナビ， ゲーム機などの家電
製品にも組み込まれています．従来，家電製品用ソフトウェアは製品ごとに開発されていましたが， OS の搭載により開発期間が短縮されています．特に，寿命が短く，販売台数の多い携帯電話では OS の存在が重要で，Symbian，iOS，Android，Windows Phone などがシェアを競っています．家電製品用の OS は，パソコンに比べると目立たない存在ですが，販売台数が圧倒的に多いため，今後の IT ビジネスではとても重要な意味を持ちます．
練習問題
PC の OS に関する記述のうち，適切なものはどれか．[平成 22 年度春期 IT パスポート試験]
(ア) OS が異なっていても OS とアプリケーションプログラム間のインタフェースは統一されているので，アプリケーションプログラムは OS の種別を意識せずに処理を
行うことができる．
(イ) OS はアプリケーションプログラムに対して， CPU やメモリ， 補助記憶装置などのコンピュータ資源を割り当てる．
(ウ) OS はファイルの文字コードを自動変換する機能をもつので， アプリケーションプ
ログラムは， ファイルにアクセスするときにファイル名や入出力データの文字コード種別の違いを意識しなくても処理できる．
(エ) アプリケーションプログラムが自由に OS の各種機能を利用できるようにするために，OS には，そのソースコードの公開が義務付けられている．
